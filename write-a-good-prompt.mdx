# Write a good prompt

A good prompt is clear, specific, and realistic.

You do **not** need technical words.

## Use this format

```text
Build a web app for [target user].

The app should help them [main job to be done].

Include these features:
1) ...
2) ...
3) ...

Important rules:
- ...
- ...

For v1, success means:
- ...
```

## What to include

- Who this app is for
- The top 3-5 features
- Any non-negotiables (for example: "must support team logins")
- What "good enough for v1" means

## What to avoid

- "Build the next unicorn"
- 20+ features in one request
- Vague words like "make it perfect"

## Good example

"Build a web app for local gyms to manage class bookings.
Users: gym owners and members.
Core features: class calendar, member booking, cancel booking, and admin dashboard.
For v1, success means owners can publish classes and members can book from mobile."

<Tip>
Start small. A tighter v1 prompt usually ships faster and better.
</Tip>
