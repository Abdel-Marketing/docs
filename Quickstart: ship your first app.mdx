# Quickstart: ship your first app

This guide is for first-time users.

Goal: get a working first version live fast, then improve it.

## Step 1) Start a new project

Open LaunchPulse and create a new project.

## Step 2) Paste a clear first prompt

Use this simple template:

```text
Build a web app for [who it is for].

Main problem:
[one sentence]

Core features:
1) ...
2) ...
3) ...

Must-have pages:
- ...
- ...

Success for v1:
- ...
```

## Step 3) Wait for the first build

Let LaunchPulse run the build.

When it finishes, open the preview and test the 2-3 most important flows.

## Step 4) Request one focused improvement

Ask for one change at a time, for example:

- "Improve the onboarding flow to 3 simple steps."
- "Add a basic admin page for managing users."

## Step 5) Repeat until v1 is usable

Keep changes small and clear.

Ship once your core flow works end-to-end.
