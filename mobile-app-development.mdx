# Mobile App Development

## Overview
Use this path when your product is app-store-first.

## When to use it
- iOS and Android launch plans
- Subscription/mobile IAP use cases

## How it works
Choose **Mobile App** at project creation. LaunchPulse follows a mobile-focused build and publishing workflow.

## Mobile vs Web (quick comparison)

| Area | Mobile App Development | Web App Development |
| --- | --- | --- |
| Primary destination | App Store / Play Store | Browser URL |
| Typical launch path | Store publish flow | Deploy + custom domain |
| Payments path in source docs | RevenueCat | Stripe |

## Key capabilities
- Mobile app build workflow
- App store publishing path
- Mobile monetisation path
