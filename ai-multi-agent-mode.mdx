# AI Multi-Agent Mode

<Warning>
Beta / in progress: capabilities and workflow details may change.
</Warning>

## Overview
AI Multi-Agent Mode is designed for larger projects where dividing work across specialist roles can improve quality and control.

## When to use this
- Complex projects with many features
- Teams that want clearer review checkpoints
- Builds where one workflow may lose context

## How it works
- Planner agent scopes the work
- Builder implements features
- Reviewer checks quality and regressions
- QA/testing validates output

## What you can do with it
- Separate planning from implementation
- Add stronger review loops
- Track role-oriented progress

## Example use cases
- Multi-module SaaS app with role-based access and billing
- Product with many cross-feature dependencies

## Example prompt
"Use Multi-Agent Mode and prioritise planning and review checkpoints before build approval."
