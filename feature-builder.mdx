# Feature Builder

## Overview
Feature Builder helps you avoid vague or bloated first builds by turning your idea into a structured plan you can edit before implementation starts.

## When to use this
- You have a big idea and need MVP scope
- You need confidence in build order
- You want more control before code generation

## How it works
- LaunchPulse suggests a build plan + feature list
- You edit priorities and remove low-value items
- Build starts only after your approval

## What you can do with it
- Keep v1 focused
- Separate must-haves from nice-to-haves
- Improve launch readiness with better sequencing

## Example use cases
- Reduce a 20-feature idea to a 5-feature MVP
- Prioritise onboarding + billing ahead of secondary features

## Example prompt
"Create a phased feature plan for a founder CRM. MVP first, then retention features."
