# Quickstart: ship your first app

## Overview
This is the fastest path for first-time users.

## Before you start

- Decide if you are building a **Web App** or **Mobile App**
- Prepare a short problem statement and 3-5 must-have features
- Keep v1 scope small

## Step-by-step guide

<Steps>
  <Step title="1) Create a new project">
    Open the Projects Dashboard and click new project.
  </Step>
  <Step title="2) Choose platform and mode">
    Pick Web App or Mobile App, then select Autonomous AI Software Engineer for your first build.
  </Step>
  <Step title="3) Write your first prompt">
    Use a simple structure:

    ```text
    Build a [web/mobile] app for [target users].
    Core problem: [one sentence].
    Must-have features:
    1) ...
    2) ...
    3) ...
    Success for v1:
    - ...
    ```
  </Step>
  <Step title="4) Review the generated plan">
    Remove low-priority features and reorder the backlog to keep MVP scope tight.
  </Step>
  <Step title="5) Approve and let it build">
    LaunchPulse runs the build workflow and produces a preview.
  </Step>
  <Step title="6) Test your core flow">
    Validate one end-to-end user journey before adding new features.
  </Step>
  <Step title="7) Iterate with focused prompts">
    Ask for one meaningful improvement at a time.
  </Step>
  <Step title="8) Deploy">
    Go live when your main flow works reliably.
  </Step>
</Steps>

## What happens next

- Add capabilities like authentication or payments
- Invite collaborators if needed
- Move into deployment workflows
