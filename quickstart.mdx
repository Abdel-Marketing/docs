# Quickstart: ship your first app

Go from blank page to a usable first version in one focused session.

**Estimated time:** 20-40 minutes for first build setup, plus iteration time.

## Before you begin

- Choose one target: **Web App** or **Mobile App**
- Prepare a short product brief (audience, problem, top 3-5 features)
- Decide your v1 success criteria (one flow that must work)

## What you'll build

By the end of this guide, you should have:

- A new LaunchPulse project
- A reviewed and approved feature plan
- A first generated app preview
- One meaningful iteration request

## Step-by-step guide

<Steps>
  <Step title="1) Create project">
    Open the Projects Dashboard and create a new project.
  </Step>
  <Step title="2) Pick platform + mode">
    Pick Web or Mobile, then choose **Autonomous AI Software Engineer** for first-time speed.
  </Step>
  <Step title="3) Submit your first prompt">

```text
Build a web app for small design studios.

Core problem:
Studios need to manage leads, proposals, and project status in one place.

Must-have features:
1) Lead pipeline
2) Proposal status tracker
3) Client portal login

Success for v1:
- Team can create leads and move them through stages
- Clients can log in and view project status
```
  </Step>
  <Step title="4) Review the generated plan">
    Remove non-essential features, reorder priorities, and keep the first release MVP-sized.
  </Step>
  <Step title="5) Approve and build">
    Start the build and wait for the first preview.
  </Step>
  <Step title="6) Validate your core flow">
    Test your most important user journey end-to-end.
  </Step>
  <Step title="7) Request one focused iteration">
    Example: "Simplify onboarding to 3 steps and reduce required fields."
  </Step>
  <Step title="8) Prepare to launch">
    Move to deployment flow once the core journey is stable.
  </Step>
</Steps>

## Common mistakes

- Asking for too many features in v1
- Skipping plan review before build starts
- Making several unrelated iteration requests at once
- Testing edge cases before the core flow works
