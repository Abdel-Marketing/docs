# What is LaunchPulse?

LaunchPulse is an AI-powered software creation platform for turning ideas into real products. Describe what you want to build, choose whether it should be a web app or mobile app, preview the result as it takes shape, and publish when it is ready.

You can use LaunchPulse to build SaaS MVPs, dashboards, internal tools, mobile apps, games, workflows, customer portals, AI-powered apps, and custom software—without starting from a blank codebase.

<Info>
  New to LaunchPulse? Start with one clear use case, ship version one fast, then improve through short prompt-driven iterations.
</Info>

## How LaunchPulse works

<Steps>
  <Step title="Start with an idea or choose one from the AI app catalog">
    Begin from your own product concept or select a promising opportunity from curated AI app ideas.
  </Step>
  <Step title="Describe what you want to build">
    Explain the user, problem, and outcomes in plain language, then set web or mobile as your build path.
  </Step>
  <Step title="Review and refine the plan">
    Confirm scope, trim complexity, and prioritize must-have features before you move forward.
  </Step>
  <Step title="Preview the product live">
    Test key screens and flows early so you can validate usability before launch.
  </Step>
  <Step title="Iterate with prompts">
    Ask LaunchPulse to improve details like feature logic, copy, layout, and flow quality.
  </Step>
  <Step title="Deploy or publish">
    Publish to web, connect a custom domain, or move through mobile publishing for iOS and Android.
  </Step>
</Steps>

## Example first prompt

```text
Build a mobile app for fitness coaches to manage clients, track workouts, send reminders, and accept monthly subscriptions.

Start with:
- coach dashboard
- client profiles
- workout plans
- booking flow
- subscription payments
- mobile-first design

Make it clean, modern, and easy for non-technical coaches to use.
```

## Good first project checklist

- Pick one clear user
- Define the main problem
- Choose web or mobile
- List 3–5 must-have features
- Keep version one simple
- Preview and test the main flow before adding more
- Publish only when the core experience works
