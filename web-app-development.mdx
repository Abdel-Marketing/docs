# Web App Development

## Overview
Use this path when your product is browser-first.

## When to use it
- SaaS dashboards
- Internal tools
- Marketplaces and portals

## How it works
Choose **Web App** at project creation. LaunchPulse builds frontend, backend, and core integrations in one workflow.

## Web vs Mobile (quick comparison)

| Area | Web App Development | Mobile App Development |
| --- | --- | --- |
| Primary destination | Browser URL | iOS / Android stores |
| Typical launch path | Deploy + custom domain | Publish to App Store / Play Store |
| Payments path in source docs | Stripe | RevenueCat |

## Key capabilities
- Auth-ready flows
- Managed storage/database support
- AI service integration
- Custom domain workflow
