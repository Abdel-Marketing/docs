# Payments & Monetisation

## Overview
LaunchPulse supports payment-ready app flows for subscriptions and paid features.

## When to use it
- Subscription pricing
- Paid feature gating
- Web or mobile monetisation setup

## How it works
Request monetisation in plain language, then LaunchPulse wires core payment and access logic.

## Provider paths

| Provider | Best fit | Platform path in source docs |
| --- | --- | --- |
| Stripe | Web app checkout/subscriptions | Web |
| RevenueCat | Mobile subscriptions/IAP | Mobile |

## Example prompts
- "Add a free plan and a Pro monthly plan."
- "Lock advanced analytics behind paid access."
- "Add monthly and annual pricing options."

## Key capabilities
- Subscription flow setup
- Plan-based feature access
- Web and mobile monetisation paths
