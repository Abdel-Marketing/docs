# Payments & Monetisation

Monetisation is part of product design, not just checkout. LaunchPulse supports web and mobile payment paths so you can validate revenue early.

## When to use this
- Subscription or paid feature models
- Revenue experiments for MVPs

## How it works
- Define plans/access model in your prompt
- LaunchPulse wires core payment and gating logic
- Validate purchase and access flows end-to-end

## Provider paths

| Platform path | Provider |
| --- | --- |
| Web | Stripe |
| Mobile | RevenueCat |

## Example prompt
```text
Add Free, Pro Monthly, and Pro Annual plans. Gate advanced analytics behind Pro.
```
