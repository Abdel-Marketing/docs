# Troubleshooting

## Overview
Use this page when builds, billing, deployment, or access workflows are not behaving as expected.

## Issue triage table

| Issue | Likely cause | Fix |
| --- | --- | --- |
| Build appears stuck | Credits depleted or interrupted run | Refresh workspace, check credits, retry from latest approved plan step |
| Preview not updating | Build still running or failed step | Check build status/logs, then refresh preview |
| iOS publish failed | Credential/team/app ID mismatch | Recheck iOS credentials and identifiers |
| Android publish failed | Service account/package mismatch | Recheck Play service account JSON and package name |
| Custom domain not live | DNS not propagated or record mismatch | Verify DNS records and wait for propagation |
| Teammate cannot access project | Missing permission/invite issue | Recheck invite and access role |

## Quick workflow

<Steps>
  <Step title="1) Reproduce exactly">
    Capture the exact action, screen, and expected result.
  </Step>
  <Step title="2) Check the matching issue row">
    Use the table above to identify likely cause.
  </Step>
  <Step title="3) Apply one fix at a time">
    Re-test after each fix.
  </Step>
  <Step title="4) Escalate with context">
    If still blocked, share project ID and clear reproduction steps with support.
  </Step>
</Steps>
