# Testing Agent

<Warning>
Beta / in progress: final behavior and controls are still being defined.
</Warning>

## Overview
Testing Agent is designed to add a dedicated QA layer for build validation and regression checks.

## When to use this
- Pre-release quality checks
- High-risk flows (auth, billing, onboarding)
- Projects with frequent iterations

## How it works
- Reads planned features
- Generates test checks
- Reports failures for follow-up fixes

## What you can do with it
- Catch issues earlier
- Reduce manual bug hunting
- Improve release confidence

## Example use cases
- Validate onboarding before launch day
- Re-test billing flow after major updates

## Example prompt
"Run testing checks on signup, login, and subscription upgrade flows."
