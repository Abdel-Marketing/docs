## Overview

The fastest path from idea to a working, deployed app in LaunchPulse V2. Follow these steps the first time you build something — they map directly to the full LaunchPulse flow.

---

## Step-by-step

### 1) Create a new project
Open the **Projects** sidebar and click **New Project**.

### 2) Pick what you're building
- **Mobile App** — for iOS / Android via App Store + Play Store
- **Web App** — for hosted web apps with a custom domain

### 3) Pick your build mode
- **Autonomous AI Software Engineer** — one capable agent handles everything (recommended for first-time users)
- **AI Multi-Agent Mode** — specialised agents collaborate (better for larger projects)

### 4) Describe your idea
Write your idea in plain language. Be specific about:
- who it's for
- the core problem it solves
- 2–3 must-have features
- tone / style if you have one

### 5) Click Create
LaunchPulse generates a **build plan** + **feature list** with priorities and dependencies.

### 6) Review and shape the plan
Before any code is written:
- ❌ Remove features you don't want yet
- 🔢 Reorder priorities (MVP first)
- ✏️ Add context (target users, constraints, examples)

### 7) Approve
Once the plan looks right, approve it and let LaunchPulse build.

### 8) Watch it build
- See **code being written** in real time
- See a **live preview** as features come online
- Quality checks run as it builds

### 9) Iterate
Prompt the agent for changes:
- "Add Google sign-in."
- "Add a $10 monthly subscription."
- "Make the onboarding shorter."

### 10) Connect what you need
Storage, database, AI services, payments — all in the same workspace.

### 11) Share with your team (optional)
Invite collaborators as **Viewer** or **Editor**.

### 12) Deploy
- **Web app** → attach a custom domain and go live
- **Mobile app** → use the Publish modal to ship to App Store / Play Store

---

## First-build tips

- Start small — get an MVP shipped before piling on features
- Use Custom Agents (Code Reviewer, QA Tester) once you're past v1
- Top up credits before a long build session so you don't get interrupted
