# Custom Agents

## Overview
Custom Agents are reusable helpers you can configure once and use repeatedly across iterations.

## When to use this
- You repeat the same instructions often
- You want consistent output formats
- You want clear role separation (builder vs reviewer)

## How it works
Create an agent with:
- Name and short purpose
- Prompt instructions
- Model/tool settings

Then call it whenever that role is needed.

## What you can do with it
- Create QA checklists on demand
- Run consistent copy/UX reviews
- Standardise product-planning outputs

## Example use cases
- QA reviewer agent
- PM scoping agent
- UI copy editing agent

## Example prompt
"Create a QA agent that checks happy path, edge cases, and auth access errors."
